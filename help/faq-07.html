JBroFuzz FAQ - Files & Directories 


Q: Fancy wording, what does JBroFuzz actually do?

This application sends malformed data on the network until the point that the application receiving the data brakes.

The term stateless simply implies that the different requests/replies being generated do not depend on previous ones made.


Q: How do I run the .jar file?

Say the file you want to run is named: JBroFuzz.jar

From the command line type the following:
>java -jar JBroFuzz.jar

You will need to have Java installed on your system. It's probably worth checking: http://www.java.sun.com


Q: So what is the motto?

If you can't fuzz with JBroFuzz you probably don't want to fuzz!

Paraphrasing Chet's ending line speech, this application tries to make fuzzing as simple and less time consuming as possible.


Q: What JBroFuzz is not claiming to be...

The automated answer to your problems regarding sending a bunch of AAA's to a listening deamon service.

Unfortunately, you still have to know a bit about the network protocol you are fuzzing, as well as exactly what type of fuzzing you want to actually perform. 


Q: How can I download the latest copy of JBroFuzz?

JBroFuzz is an OWASP Project. It is free to download and open source.

To make sure you are running the latest version select "Check for Updates..." from the Options menu. 


Q: How do I install JBroFuzz?

There is no installation requirement. The application is a single .jar file. There is also an executable .exe for win32 users.

You will need to have Java installed on your system. It's probably worth checking: http://www.java.sun.com  


Q: What version(s) of Java does JBroFuzz support?

JDK/JRE 1.6 and later.

Q: What are all these folders that get created when I run the JBroFuzz?

JBroFuzz creates the following 3 folders under the folder jbrofuzz in the location that it got launched from:

  • fuzzing
  • sniffing
  • web-dir

Each one is used for the corresponding tab.


Q: I run the exe or jar and the folder jbrofuzz does not get created, what's wrong?

For most browsers, if you select to run the exe / jar after it has finished downloading through the web browser, you will be limited to the security policy in place.

Typically, this will not allow the application to create any files or folders within the downloaded location.

Run the exe or jar file through the operating system, instead of the browser.

Q: What is a generator?

A generator contains a particular character set which will be tested against the specified range of the request. As this might sound pretty Greek to most, here are some examples of generators: Sending all requests of : 0000 to 1111 (binary generator over 4 characters) Sending all requests of : 00 to 99 (decimal generator over 2 characters).

To get a better understanding of how generators work, view the tutorial demo that is available for download..

Q: What is a fuzzer?

A fuzzer is an object that iterates through a set of values grouped in a generator. The values attached in a fuzzer represent the payloads of the particular generator.

To get a better understanding of how fuzzers work, unjar the latest version of JBroFuzz and look at the file generators.jbrofuzz .

Q: Why can't I add a generator?

You need to select the part of the request that you would like to fuzz.

This then gives you the ability to add a number of generators on the specified range.

A s a result, the generator gets added to the generators list on the right hand side.

Q: Does JBroFuzz support UDP?

That would be a No.

Q: While 'sniffing' the process fails to start?

Are you sure you don't have another service listening on the given port?

Try a different port, ideally above 1024, on which a standard service is not operational.
