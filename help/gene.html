Generators Tab 

The generators tab provides a list of all generators that have been successfully loaded during start-up.

 

Generators are used for fuzzing. By selecting part of the message request in the "TCP Fuzzing" tab you can add a generator to a specific range of the request, thus fuzzing that range.

How to use

  • This tab offers information on successfully loaded generators
  • The list provided is available for use within the "Add Generator" functionality of the "TCP Fuzzing" tab 

What to expect

A list of all successfully loaded generators is diplayed at the top. This is followed by all generator details. These are in the following format:

  • Each generator has a three letter codename (e.g. INT, DEC)
  • Each generator has a name that follows (e.g. Integer Overflows)
  • The corresponding tests follow:
    • One test per line
    • Generator ends by a blank line

Advanced options

You can specify you own generators to be loaded during startup, by modifying the jbrofuzz-generators file.

The particular file format used within the file should be followed.

  • # at the beginning of a line indicates a comment line 
      

jbrofuzz-generators file:

This file is distributed with jbrofuzz.

 

The application expects the file to be in the same directory as that from which jbrofuzz is launched from.

 

An identical file exists within the bundled jar/exe file. In the event of not being able to locate the jbrofuzz-generators file in the launch directory, the file within the jar/exe will be used. 

 

Not being able to locate the jbrofuzz-generators file will generate a note in the "System" tab, stating that the file could not be found.
