TCP SniffingTab 

This tab allows you to act as a TCP proxy between two particular locations.

A TCP proxy acts as a "man-in-the-middle" relaying all traffic. With this functionality available in the sniffing tab, you can assess how vulnerable a particular application or protocol is vulnerable to this type of attack.

This is achieved by examining the information flowing between the two particular locations.

How to use  

The following information is required in order to start a TCP sniffing session:

  • Remote Host (either IP Address, or resolvable DNS name) 
  • Remote Port (1-65535)
  • Local Host (either IP Address, or resolvable DNS name) 
  • Local Port (1-65535)

Supplying the above information and selecting "Start" will enable you to snif any traffic flowing between the remote and local host.

For this, you would have to generate some traffic on the corresponding local host and port connection.

What to expect

A list of rows will begin to be populated in the Output panel. Each file contains the data travelling between the remote host and local host.

Use your mouse to select a row in order to display the contents of the data.

Advanced options

You can launch your browser, with the corresponding proxy settings in place.

  • Select "Bro" to launch your browser 
       

Logging & Saving:

All TCP sniffing session data is saved within the directory:

  • jbrofuzz
    • sniffing

Within this directory a session directory matching the date and time of launch of jbrofuzz is created. A typical format of this is:

 

/203 2007-07-22 00-05-15/

 

Data is saved in a file of increasing number. A typical format of this is:

 

01-000001.html

 

Launching your browser with "Bro":

This function aims to allow you to simply launch your web browser, targeting the host in question.
