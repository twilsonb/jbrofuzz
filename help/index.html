Help Topics 

JBroFuzz is a stateless network protocol fuzzer that emerged from the needs of penetration testing. JBroFuzz's design emphasizes simplicity and ease-of-use. We are attempting to get you to cover the basics in terms of penetration testing for web applications by means of creating malformed data and having the network protocol in question consume the data.

The one-page summary here within aims to provide all the information for you to get started and use JBroFuzz productively. For more detailed information, you can also use the left-hand menu to navigate to particular topics.

A Brief Overview of JBroFuzz

JBroFuzz consists of a number of tabs. You can navigate through the tabs by clicking on each one located at the bottom left corner within the window of the application.

You can also show or hide individual tabs by using the "Show/Hide" menu and then selecting the corresponding tab. 

By default four tabs are shown:

  • Fuzzing Allows you to perform plaintext fuzzing operations over HTTP or HTTPS, attaching any number of generators to the request being made
  • Sniffing Allows you to act as a TCP proxy between any two locations by establishing a man in the middle channel
  • Web Directories Allows you to perform directory enumeration attacks over HTTP or HTTPS

There are also two more tabs:

  • Payloads Shows the list of available loaded generators containing templates for XSS, SQL injection and other attacks 
  • System Provides useful information regarding system operations 

Fuzzing Concepts: Fundamentals

Fuzzing describes a methodology for software testing.

 

The concept stems from Boundary Value Analysis (BVA) [1]

 

"...Fuzzing can take many forms depending on the type of protocol, and the desired tests..." [2]

 

There are many fuzzer types [3]. JBroFuzz attempts to encompass the following category types: 

  • Network Protocol Fuzzer
  • Web Application Fuzzer
