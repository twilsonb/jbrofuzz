TCP Fuzzing Tab 

This tab allows you to perform fuzzing operations over TCP.

A number of generators can be added to the message request being submitted. This process is refered to as fuzzing. The generators used are the generators available within the generators tab. These are read from file during start-up. 

How to use (Basic) 

The following information is required in order to start a TCP fuzzing session:

  • Target (either IP Address, or resolvable DNS name) 
  • Port (1-65535)
  • Request (message transmitted)

Supplying the above information and selecting "Fuzz" will sent a single request to the target in question, over the specified port, with the corresponding message supplied.

How to use (Advanced) 

In order to perform fuzzing over TCP, you will need to add one or more generators to the request being submitted:

  • Select (highlight) part of the message request 
  • Select "Add a Generator" 
  • Select the type (e.g. DEC for "Decimal Fuzz Type")

As an example, in the case you selected the above and highlighted a single character from the request, a total of 10 requests iterating through digits 0 to 9 will be sent to the target.

  • Select "Fuzz!" to begin  

What to expect

A list of rows will begin to be populated in the Output panel. Each file contains the request made followed by the reply received.

Use your mouse to select a row in order to display the contents of the request submitted and the reply received.

Advanced options

Once one or more requests have been submitted and requests received, JBroFuzz gives you the option to plot the results obtains.

  • Select "Bro" to plot the results obtains 
      

Logging & Saving:

All TCP fuzzing session data is saved within the directory:

  • jbrofuzz
    • fuzzing

Within this directory a session directory matching the date and time of launch of jbrofuzz is created. A typical format of this is:

 

/203 2007-07-22 00-05-15/

 

Each request and reply is saved in a file of increasing number. A typical format of this is:

 

01-000001.html

 

Plotting with "Bro":

This function aims to capture differences in responses.

Each request/response is hashed to a value between [0 - 1000].

This is achieved by recursively adding the corresponding bytes in the file.

 

As a result, you can observe minor differences (e.g. an timestamp change) as well as major differences (e.g. actually receiving a different response.

All values are compared to the first request/response that was triggered.
